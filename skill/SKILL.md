# 汽车之家车型口碑采集

按下面流程执行，目标是稳定抓取汽车之家车型口碑，并输出可复核、可分析的 Excel。

## 1. 确认输入

优先接受三种输入：

- 完整口碑页 URL
- `seriesId + dimensionid + 页码范围`
- 自然语言任务描述（如“抓风云X3L最满意和最不满意 1-13 页导出 Excel”）

已确认的维度规则：

- `dimensionid=10` → 最满意
- `dimensionid=11` → 最不满意

分页规则：

- 第 1 页：`https://k.autohome.com.cn/{seriesId}?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`
- 第 n 页：`https://k.autohome.com.cn/{seriesId}/index_{n}.html?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`

页码处理规则：

- 如果用户明确给了页码范围，就按用户指定页码抓取
- 如果用户没有给页码，或明确说“全部页面 / 所有页面 / 所有条目”，则先检测该车型口碑分页总页数，再按 `1 ~ 最后一页` 抓取
- 若用户给了 `start-page` 但没给 `end-page`，脚本默认会自动探测 `end-page`

## 2. 抓取方式

优先使用浏览器方式读取页面可见文本，不要默认走 OCR。

执行原则：

- 先打开列表页
- 从列表页提取结构字段
- 评价详情先取列表页展示文本
- 正式开发或高精度任务前，抽样对比详情页正文是否更完整
- 若详情页明显更完整，则改为“列表页取结构字段 + 详情页补全文”

## 3. 字段清单

输出以下 18 个字段：

1. 口碑维度
2. 数据类型
3. 用户名
4. 发表日期
5. 口碑标题
6. 综合口碑
7. 车型
8. 行驶里程
9. 电耗
10. 裸车购买价
11. 参考价格
12. 购买时间
13. 探店时间
14. 购买地点
15. 探店地点
16. 评价详情
17. 来源链接
18. 抓取页码

字段原则：

- `评价详情` 必须保留网页原文，不删改、不摘要、不纠错
- `电耗` 保留页面原文，如“14.6kWh 春秋电耗”
- 页面无字段时留空，不强行补值
- 价格 / 时间 / 地点必须拆列，不再混到同一列

## 4. 用户名规范化

不要删除“认证”信息。

如果页面展示为：
- `真实用户名 风云X3L认证`

则规范化为：
- `真实用户名_风云X3L认证`

普通 `真实用户名 风云X3L` 形式：
- 可保留原样，或仅去掉车型尾缀
- 不得改动用户名本体

## 5. 数据类型判定

同时兼容两类记录：

- `车主购车口碑`
- `试驾探店口碑`

满足任一条件时，判为 `试驾探店口碑`：

- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 正文前缀为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`。

不要丢弃试驾/探店数据，除非用户明确要求只保留真实购车口碑。

## 6. 导出规则

默认文件命名：

- `ZJ+车型名称+最满意or最不满意_页数范围.xlsx`
- 例如：`ZJ风云X3L最满意or最不满意_1-13页.xlsx`
- 若未显式指定页数范围，可省略页数后缀

当前脚本的实际输出是“按来源链接对齐后的双 sheet”结构，也是当前对外唯一交付形态：

- `购车口碑`
- `试驾口碑`

每行同时保留：

- `最满意`
- `最不满意`
- `来源链接`
- `最满意抓取页码`
- `最不满意抓取页码`

如需调整输出契约，需要同步改脚本、README 和下游契约。

## 7. 校验规则

每条记录至少应包含以下核心字段：

- 用户名
- 综合口碑
- 车型
- 来源链接

缺任意一项时：

- 标记为异常记录
- 不直接写入正式结果
- 记录到异常清单或 JSON 侧产物中

每页抓取后检查：

- 是否成功拿到口碑列表区
- 是否识别到合理数量的口碑卡片
- 是否出现大面积字段为空

正式导出后，必须再做一层**结果级校验**：

- 检查 `最满意` 总数是否等于 `最不满意` 总数
- 检查同一页的 `最满意` / `最不满意` 条数是否一致
- 生成同名 `.validation.json` 报告，保留分页计数与异常原因

处理原则：

- 若只是普通导出：允许输出 Excel，但必须明确提示用户校验失败
- 若使用 `--strict-validate`：总数或分页数量不一致时，脚本必须返回非零退出码，不能把有问题的结果当成成功
- 发现类似“某一页整页漏写”这类问题时，要优先提示用户复跑，不要默认认为站点真实数据就是不一致

## 8. 重试规则

分页抓取失败时：

- 自动重试 2 次
- 连续失败后停止静默继续，明确报告页码和错误原因

常见瞬时错误包括：

- `ERR_ABORTED`
- 页面导航中断
- 页面内容未加载完整

## 9. 实操提醒

- 不要按卡片内部的子 `listitem` 粗暴切分记录
- 必须按顶层口碑卡片切分
- 不要因正文口语化、像广告、像生成式措辞就擅自删除，只处理明显不属于正文区域的解析噪音
- 批量任务除 Excel 外，最好同步保留 JSON 原始结构化结果，方便回溯和修规则
- 正式开发前，至少抽样对比 5 条列表页正文与详情页正文

## 12. 附带脚本

本 skill 附带脚本：

- `scripts/export_autohome_koubei.py`

用途：
- 按字段清单导出 Excel
- 默认支持最满意 / 最不满意双维度抓取
- 当前实现固定导出 2 sheet 对齐结构
- 导出后自动生成同名 `.validation.json` 校验报告，检查最满意 / 最不满意总数与分页条数是否一致

## 13. 进度输出

脚本支持三类进度输出：

- `--progress`：终端显示进度条
- `--progress-file /path/to/job.progress.json`：写进度 JSON，适合对话框或前端轮询
- `--progress-webhook https://...`：推送到通用 webhook
- `--feishu-webhook https://...`：直接推送到飞书 incoming webhook，机器人启用安全密钥时可配 `--feishu-secret`

采集层的进度粒度按阶段和页数计算：

- 自动探测总页数
- 采集 `最满意`
- 采集 `最不满意`
- 写出 Excel
- 写出校验报告

对话框侧最小轮询示例：

```python
@app.get("/api/jobs/{job_id}/progress")
def job_progress(job_id):
    return json.loads(Path(progress_path).read_text(encoding="utf-8"))
```

前端侧每 1-2 秒轮询一次，读取 `percent`、`stage`、`message` 即可。
- 可通过 `--strict-validate` 在校验失败时直接报错退出，避免静默产出错误文件

当前实现的关键契约：

- 采集层输出固定为 2 sheet 对齐表
- `来源链接` 是两侧记录的对齐键
- `最满意` / `最不满意` 直接作为下游摘要输入

示例：

```bash
python3 skills/auto-koubei-collector/scripts/export_autohome_koubei.py \
  --series-id 8208 \
  --start-page 1 \
  --end-page 13 \
  --output ./ZJ风云X3L最满意or最不满意v2.xlsx \
  --workdir /Users/xyc/.openclaw/workspace
```

如果要自动探测总页数再抓取，可省略 `--end-page`：

```bash
python3 skills/auto-koubei-collector/scripts/export_autohome_koubei.py \
  --series-id 8208 \
  --start-page 1 \
  --auto-detect-pages \
  --output ./ZJ风云X3L最满意or最不满意_全量.xlsx \
  --workdir /Users/xyc/.openclaw/workspace
```

如需在校验失败时直接报错退出，追加：

```bash
python3 skills/auto-koubei-collector/scripts/export_autohome_koubei.py \
  --series-id 8208 \
  --start-page 1 \
  --end-page 13 \
  --output ./ZJ风云X3L最满意or最不满意v2.xlsx \
  --workdir /Users/xyc/.openclaw/workspace \
  --strict-validate
```

## 12. 推荐调用方式

优先按下面顺序接收输入：

1. **直接给汽车之家口碑页 URL**
2. **直接给车型 `seriesId`**
3. **给明确车型名，并允许先定位车型页面**

推荐示例：

- 帮我抓这个汽车之家口碑页对应车型的最满意和最不满意，导出 Excel：`https://k.autohome.com.cn/8208?dimensionid=10&order=0&yearid=0#listcontainer`
- 帮我抓 8208 最满意和最不满意口碑，导出 Excel
- 帮我抓汽车之家风云T11最满意和最不满意口碑，先帮我定位车型页面，再导出 Excel

说明：

- 可以只用自然语言调用
- 但当前版本最稳的方式仍然是：**提供口碑页 URL 或 seriesId**
- 若只给车型名，也可以执行，但通常需要先定位该车型在汽车之家的口碑页面
