# 汽车之家车型口碑采集规则参考

## 1. URL 规则

### 维度参数
- `dimensionid=10` → 最满意
- `dimensionid=11` → 最不满意

### 分页规则
- 第 1 页：`https://k.autohome.com.cn/{seriesId}?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`
- 第 n 页：`https://k.autohome.com.cn/{seriesId}/index_{n}.html?dimensionid={dimensionid}&order=0&yearid=0#listcontainer`

## 2. 文件命名规则

默认命名：
- `ZJ+车型名称+最满意or最不满意_页数范围.xlsx`

例如：
- `ZJ风云X3L最满意or最不满意_1-13页.xlsx`
- `ZJ风云T11最满意or最不满意_1-3页.xlsx`

## 3. 导出规则

正式版默认按“维度 × 数据类型”拆成 4 个 sheet：
- `最满意_车主购车口碑`
- `最满意_试驾探店口碑`
- `最不满意_车主购车口碑`
- `最不满意_试驾探店口碑`

若用户明确要求简化，可退化为：
- `最满意`
- `最不满意`

## 4. 用户名规范化规则

- 不删除“认证”信息
- 若页面展示为：`真实用户名 风云X3L认证`
- 规范化输出为：`真实用户名_风云X3L认证`
- 普通 `真实用户名 风云X3L` 形式，可仅去掉车型尾缀
- 不得改动用户名本体

## 5. 数据类型判定规则

判为 `试驾探店口碑` 的信号：
- 出现 `参考价格`
- 出现 `探店时间`
- 出现 `探店地点`
- 正文前缀为 `好评` 或 `槽点`
- 标题或正文明确出现 `试驾` 或 `探店`

否则默认判为 `车主购车口碑`

## 6. 正式版字段

1. 口碑维度
2. 数据类型
3. 用户名
4. 发表日期
5. 口碑标题
6. 综合口碑
7. 车型
8. 行驶里程
9. 电耗
10. 裸车购买价
11. 参考价格
12. 购买时间
13. 探店时间
14. 购买地点
15. 探店地点
16. 评价详情
17. 来源链接
18. 抓取页码

## 7. 校验规则

每条记录至少包含：
- 用户名
- 综合口碑
- 车型
- 来源链接

缺任一项时：
- 标记异常
- 不直接写入正式结果
